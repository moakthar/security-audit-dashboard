# Security Audit Log Investigation Dashboard

A full-stack web application that enables security engineers to upload, search, filter, sort, paginate, and investigate system audit logs efficiently.

---

## Project Overview

The **Security Audit Log Investigation Dashboard** is designed to help security teams analyze and investigate audit logs generated by enterprise applications.

The application supports:

* Bulk upload of audit logs
* Advanced search
* Multi-criteria filtering
* Server-side sorting
* Pagination
* Responsive dashboard
* Efficient querying using MongoDB indexes

## Audit Log Schema

```json
{
  "actor": "priya.nair@company.com",
  "role": "admin",
  "action": "DELETE_USER",
  "resource": "/api/users/334",
  "resourceType": "USER",
  "ipAddress": "192.168.1.45",
  "region": "ap-south-1",
  "severity": "HIGH",
  "status": "Unresolved",
  "timestamp": "2025-06-14T08:32:11Z"
}
```

---

## REST API

### Upload Logs

```
POST /api/logs/upload
```

Uploads audit log records into MongoDB.

---

### Get Logs

```
GET /api/logs
```

Supports:

* Search
* Filtering
* Sorting
* Pagination

Example:

```
GET /api/logs?page=1&limit=20&search=admin&role=admin&severity=HIGH&status=Resolved&region=ap-south-1&sortBy=timestamp&order=desc
```

---

### Get Single Log

```
GET /api/logs/:id
```

---

### Delete Log (Optional)

```
DELETE /api/logs/:id
```

## Technical Decisions

* Express.js REST API architecture
* Mongoose schema validation
* MongoDB indexes for fast querying
* Server-side pagination to reduce payload size
* Bulk insertion using `insertMany()` for efficient uploads
* Modular folder structure for scalability and maintainability
* Separation of concerns using controllers, services, middleware, and validators

---

## Performance Considerations

* Bulk insert optimized using MongoDB `insertMany()`
* Indexed fields improve query performance
* Server-side pagination minimizes network traffic
* Filtering and sorting executed within MongoDB
* Lean queries (`.lean()`) used where appropriate to reduce memory overhead

---

## Assumptions

* Uploaded audit logs follow the specified schema.
* Timestamp values are in ISO 8601 format.
* Authentication and authorization are outside the scope of this assessment.
* MongoDB is running locally or via MongoDB Atlas.
